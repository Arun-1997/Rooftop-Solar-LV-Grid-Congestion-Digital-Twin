# P0.5 — explore: profile & visualise the ingest

An intermediate, **read-only** step between [P0 ingest](P0.md) and P1 (roof-plane
extraction). It opens the artefacts P0 wrote, computes summary statistics, and
renders a small figure set + a standalone HTML report — so you can sanity-check
coverage and understand the study area before any modelling. It never hits the
network.

## What it does

`rsgt explore` reads, for the AOI named in the run config:

| Input (from P0) | Insight |
|---|---|
| `bag/<aoi>_*pand*.gpkg` | building count, footprint-area distribution, construction-year/decade histogram, pand status |
| `bag3d/<aoi>_buildings.txt` | number of 3D BAG building models available for roof extraction |
| `pc6/<aoi>_*.gpkg` | PC6 polygon count + area distribution (the aggregation units) |
| `ahn/<aoi>_dsm.tif`, `_dtm.tif` | raster coverage + elevation stats, and **object height** (DSM − DTM) |
| `capacity/<aoi>_capacity_*.gpkg` | feature counts per layer (if the capacity source was enabled) |
| `interim/<aoi>_aoi.geojson` | AOI extent, area, resolution source |
| `processed/ingest_summary.json`, `raw/manifest.json` | per-source status, download count + bytes |

### Outputs — under `data/processed/explore/`

- **`report.html`** — a standalone visual report (figures embedded inline, no
  external assets) with a "key findings" summary and a link to the interactive map.
- **`map.html`** — an **interactive** Leaflet map (via folium): toggleable layers for
  BAG buildings shaded by construction year (with tooltips), PC6 units, the AOI
  outline, and any capacity layers. The geometry is embedded in the file; the
  *basemap tiles* are fetched from OpenStreetMap, so opening it needs an internet
  connection. Footprints are sampled above `--max-map-buildings` (default 8000) to
  keep the file responsive.
- **`insights.json`** — the same numbers as a machine-readable
  [`ExploreProfile`](../src/rsgt/explore/profile.py).
- **`figures/`** — `overview.png`, `construction_years.png`, `footprint_areas.png`,
  `ahn_dsm.png`, `ahn_object_height.png`.

AHN rasters are read **decimated** (default ≤ 1200 px on the long side): a 0.5 m
coverage of a whole municipality is far too large to hold in memory, and overview
stats and previews don't need full resolution. Tune with `--max-raster-dim`.

Each artefact and each figure is best-effort: anything missing or broken is recorded
in `load_status` and the rest of the report is still produced — the same per-item
isolation P0 uses for sources.

## Install

Needs the geospatial stack **and** the new `viz` extra (matplotlib):

```bash
pip install -e ".[geo,viz,dev]"
```

## Run

```bash
# after `rsgt ingest -c configs/oudewater.yaml`
rsgt explore -c configs/oudewater.yaml

# or the standalone script (prints a human-readable summary + the report path)
python scripts/explore.py -c configs/oudewater.yaml -v

# JSON insights only (skip the static figures / the interactive map), or a lighter raster read
rsgt explore -c configs/example_small.yaml --no-plots --no-map
rsgt explore -c configs/oudewater.yaml --max-raster-dim 800
```

(`python -m rsgt explore ...` works identically.)

## Tests
