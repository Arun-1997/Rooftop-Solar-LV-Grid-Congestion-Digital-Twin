# P0 — AOI + data ingest + reproject

Phase zero of the [project spec](../README.md#8-phased-plan-iterative-depth):
pick one small municipality, pull every open dataset that intersects it, and land
it all in RD New, cached and reproducible.

## What it does

`rsgt` Stage 0 resolves an **area of interest** and ingests, for that AOI:

| Source | Service | Output |
|---|---|---|
| **3D BAG** LoD2.2 | OGC API Features `api.3dbag.nl` (CityJSONFeatures, EPSG:7415) | `bag3d/<aoi>.city.jsonl` (CityJSONSeq) + `<aoi>_buildings.txt` |
| **AHN** DSM/DTM | PDOK WCS 2.0.1 (`dsm_05m` / `dtm_05m`) | `ahn/<aoi>_dsm.tif`, `<aoi>_dtm.tif` (EPSG:28992) |
| **BAG** attributes | PDOK WFS `bag:pand` | `bag/<aoi>_bag_pand.gpkg` |
| **PC6** polygons | PDOK/CBS WFS `postcode6:postcode6` | `pc6/<aoi>_postcode6_postcode6.gpkg` |
| **Capacity map** | Netbeheer ArcGIS FeatureServer *(config-driven)* | `capacity/<aoi>_capacity_*.gpkg` |
| **AOI boundary** | PDOK OGC API `gemeentegebied` | `interim/<aoi>_aoi.geojson` |

Everything is reprojected to / requested in **EPSG:28992 (RD New)** (3D BAG stays
in its native **EPSG:7415**, which shares the planar axes). Every download is
recorded in `data/raw/manifest.json` (URL, params, sha256, size, timestamp) so a
re-run is a cache hit, and a run summary is written to
`data/processed/ingest_summary.json`.

## Install

P0 needs the geospatial stack (the `geo` extra):

```bash
pip install -e ".[geo,dev]"
```

## Run

```bash
# fast smoke run: a ~300 m block in Oudewater (seconds)
rsgt ingest -c configs/example_small.yaml -vv

# the real P0 study area: the whole municipality of Oudewater
rsgt ingest -c configs/oudewater.yaml -vv

# just resolve + save the AOI, or restrict to some sources / force re-download
rsgt aoi    -c configs/oudewater.yaml
rsgt ingest -c configs/oudewater.yaml --only bag3d ahn --force
```

(`python -m rsgt ...` works identically.)

## Choosing the AOI

Set it in the run config (`configs/*.yaml`):

- `aoi.municipality: Oudewater` — resolved from the official boundary, or
- `aoi.bbox: [minx, miny, maxx, maxy]` — an explicit RD New box, or both
  (boundary clipped to the box). `aoi.buffer_m` pads the result.

To study a different town, copy `configs/oudewater.yaml` and change `aoi`.

## The capacity map (deliberately off by default)

The Netbeheer Nederland *capaciteitskaart* is the project's congestion **ground
truth** and must stay **held-out external validation** — never a training target
(see the spec's leakage note). Its ArcGIS FeatureServer URL is not a static link:
open <https://capaciteitskaart.netbeheernederland.nl>, find a `.../FeatureServer`
request in the browser network tab, and set `sources.capacity.feature_server` plus
the `invoeding_layer` / `afname_layer` ids in the config. Until then the source is
skipped with an explanatory status (not an error).
